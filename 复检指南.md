**修复 3：补齐 recallDebug 的 debugInfo 字段**

问题：

前端已经读取：

```text
debugInfo.queryEnhance
debugInfo.excludedPaths
debugInfo.contextBytes
debugInfo.maxContextBytes
```

但后端当前返回没有这些字段。

建议：

在 `/kmc/knowledgeBase/recallDebug` 后端响应里补字段。最小兼容结构：

```json
{
  "debugInfo": {
    "queryEnhance": {
      "strategy": "rewrite",
      "originalQuery": "总评方案是什么？",
      "variants": []
    },
    "excludedPaths": [],
    "contextBytes": 8421,
    "maxContextBytes": 12000
  }
}
```

字段来源建议：

- `queryEnhance.strategy`：从当前 query-transform 配置或实际执行策略取。
- `originalQuery`：原始 query。
- `variants`：multi_query/hyde 变体，没有就空数组。
- `excludedPaths`：没有弱路径排除就空数组。
- `contextBytes`：用最终 `contextPreview` 的 UTF-8 字节数即可，最小可验收。
- `maxContextBytes`：从 RAG context 配置取；没有就用当前默认 12000。

不要让前端自己猜这些字段。后端补最直接。

验收：
