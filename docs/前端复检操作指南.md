# qKnow 前端优化复检操作指南

> 基于本次对话中所有优化项，逐项列出前端复检操作步骤。
> 每项包含：检查位置、操作步骤、预期结果、验证命令。

---

## 一、工具管理页面（views/kb/tool/index.vue）

### 1.1 工具来源列

**检查位置**：`frontend/src/views/kb/tool/index.vue`

**操作步骤**：
1. 启动前端 `npm run dev`
2. 登录系统，进入「工具管理」页面
3. 查看表格是否有「来源」列
4. 检查每个工具行是否显示 MCP/自定义/内置 标签

**预期结果**：
- 表格显示「来源」列（第 3 列）
- MCP 工具显示蓝色 `MCP` 标签
- 自定义工具显示绿色 `自定义` 标签
- 内置工具显示灰色 `内置` 标签

**验证命令**：
```bash
grep "key: 6" frontend/src/views/kb/tool/index.vue  # 应显示来源列配置
grep "source === 'mcp'" frontend/src/views/kb/tool/index.vue  # 应显示 MCP 标签逻辑
```

---

### 1.2 MCP Server 状态面板

**检查位置**：`frontend/src/views/kb/tool/components/McpServerPanel.vue`

**操作步骤**：
1. 进入「工具管理」页面
2. 查看页面顶部是否有「MCP Server 状态」卡片
3. 点击刷新按钮
4. 检查是否有已注册的 MCP Server 列表

**预期结果**：
- 页面顶部显示 MCP Server 状态卡片
- 卡片内显示已注册的 MCP Server 列表
- 每个 Server 显示连接状态（绿色/红色指示灯）
- HTTP/Stdio 类型标签
- 刷新按钮可点击

**验证命令**：
```bash
curl -s http://localhost:8099/kb/health | jq '.data.mcp'  # 应返回 MCP 状态
```

## 二、Agent 调试面板（views/kb/agent/）

### 2.1 工具调用消息展示

**检查位置**：`frontend/src/views/kb/agent/components/MessageList.vue`

**操作步骤**：
1. 进入「Bot 管理」→ 选择一个 Bot → 进入调试页面
2. 发送一个会触发工具调用的问题（如搜索类问题）
3. 观察消息列表中是否出现工具调用卡片
4. 点击工具调用卡片的展开/折叠按钮

**预期结果**：
- 工具调用消息显示为绿色头像卡片
- 卡片头部显示工具名称 + 状态标签（成功/失败/执行中）
- 点击可展开查看输入 JSON 和输出 JSON
- 显示耗时（ms）

**验证命令**：
```bash
grep "tool_call" frontend/src/views/kb/agent/components/MessageList.vue  # 应有工具调用模板
grep "formatJson" frontend/src/views/kb/agent/components/MessageList.vue  # 应有 JSON 格式化函数
```

## 九、后端 API 端点验证

### 9.1 所有新增 API 端点

| 端点 | 方法 | 用途 | 验证命令 |
|------|------|------|---------|
| `/kb/health` | GET | 健康检查（含 MCP + LangFuse） | `curl -s http://localhost:8099/kb/health \| jq '.'` |
