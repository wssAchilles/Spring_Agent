## 二、分领域深度分析

### 2.1 RAG 引擎

#### 黄金三角：模板分块 + 混合检索 + Cross-Encoder 重排序

**RAGFlow DeepDoc 实现**（83.7k⭐）：
- 布局识别器基于 YOLOv10，识别 10 种文档组件：Text/Title/Figure/Table/Header/Footer/Reference/Equation/Code/TOC
- 表格结构识别（TSR）使用 5 标签：Column/Row/Column header/Projected row header/Spanning cell
- 模板分块：按文档类型（PDF/Word/代码/表格）选择不同分块算法
- 多路召回 + 融合重排序

**分块策略对比**（已验证数据）：
| 策略 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| 固定大小 | 简单、可预测 | 语义断裂 | 通用文本 |
| 递归分块 | 尊重文档结构 | 配置复杂 | 结构化文档 |
| 语义分块 | 语义完整性好 | 计算成本高 | 高质量需求 |
| 模板分块 | 按文档类型优化 | 需要模板库 | 企业文档 |

**检索融合**（已验证数据）：
- Hybrid RRF + Cross-Encoder Rerank：Recall@5 = 0.816（+17.4%），MRR@3 = 0.605（+39.7%），p<0.001
- HyDE 在含数字查询中适得其反：R@5 = 0.544 vs Dense = 0.587
- 弱检索路径拖低整体精度：强 FTS 0.650 + 弱 DVS 0.390 → 融合 0.604

**qKnow 优化建议**：
1. **短期**：接入 Cross-Encoder Rerank（DashScope 或自建）
2. **中期**：实现模板分块（按文档类型选择算法）
3. **长期**：DeepDoc 式文档理解（OCR + 布局分析）

#### RAG 评估指标（RAGAS v0.4.3）

| 指标 | 计算方法 | 目标值 |
|------|---------|--------|
| Faithfulness | LLM 判断答案是否有检索支撑 | >= 0.8 |
| Context Precision | 检索内容与问题相关性 | >= 0.7 |
| Answer Relevancy | 回答与问题的切题程度 | >= 0.8 |
| Context Recall | 检索是否覆盖所有相关信息 | >= 0.7 |

### 2.2 Agent 编排

#### 三大框架对比

| 维度 | LangGraph | Dify | CrewAI |
|------|-----------|------|--------|
| Stars | 35.8k | 147k | 54.4k |
| 核心理念 | 有状态图执行 | 可视化 DSL | 角色化协作 |
| 记忆系统 | 短期+长期 | 会话级 | 复合评分公式 |
| 多 Agent | Supervisor/Worker | 工作流节点 | Crews + Flows |
| Human-in-loop | 原生支持 | 审批节点 | 有限 |

#### CrewAI 记忆公式（已验证）

```
composite = 0.5 × similarity + 0.3 × decay + 0.2 × importance
decay = 0.5 ^ (age / 30)  # 30 天半衰期
consolidation_threshold = 0.85  # 超过此阈值触发记忆整合
```

**qKnow 借鉴**：
- 当前 MemoryManager 使用简单滑动窗口
- 建议引入复合评分公式，增加 importance 和 decay 维度
- 实现记忆整合（consolidation）机制

#### LangGraph 五大核心能力

1. **Durable Execution**：Agent 状态持久化，故障自动恢复
2. **Human-in-the-loop**：执行中可插入人工审批
3. **Comprehensive Memory**：短期工作记忆 + 长期持久记忆
4. **LangSmith Debugging**：执行路径可视化追踪
5. **Production-ready Deployment**：可扩展部署方案

**qKnow 借鉴**：
- 当前 DAG 执行器缺少持久化和故障恢复
- 建议实现 Agent 状态快照 + 断点续跑

### 2.5 PgVector 优化

#### PgVector v0.8 新特性

| 特性 | 说明 | 收益 |
|------|------|------|
| Iterative Index Scans | 自动扩展扫描范围 | 召回率提升 |
| Binary Quantization | 索引体积降低 32x | 存储优化 |
| Sparse Vectors | 稀疏向量支持 | 混合检索 |
| Half-precision | 半精度浮点 | 性能提升 |

**qKnow 优化建议**：
- 从 IVFFlat 迁移到 HNSW 索引
- 启用 Binary Quantization 降低存储
- 使用 Iterative Index Scans 提升召回率
