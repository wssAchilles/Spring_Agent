## 三、代码编辑器统一

### 当前问题
3 套编辑器并存：Monaco 0.55.1 + CodeMirror 5/6 + Ace 1.38

### 推荐方案：仅保留 Monaco

**理由**：
- Monaco 是 VS Code 同款引擎，功能最全
- 支持 Language Server Protocol（LSP）
- 支持 Diff Editor（用于代码对比）
- Vue 集成：`monaco-editor-vue3`（168⭐）

**执行步骤**：
1. 搜索所有使用 CodeMirror/Ace 的组件
2. 迁移到 Monaco Editor
3. 从 `package.json` 移除 `codemirror`、`@codemirror/*`、`ace-builds`、`vue3-ace-editor`、`vue-codemirror`
4. 验证构建体积减少

**工作量**：1 天
**收益**：构建体积减少 ~200KB

## 四、AI 对话界面最佳实践

### 4.1 国际顶级产品 UI 参考

| 产品 | 核心 UI 模式 | qKnow 可借鉴 |
|------|-------------|-------------|
| **ChatGPT** | 流式 Markdown + 代码高亮 + 工具调用折叠 | 已有基础，需增强工具调用展示 |
| **Claude** | 长文本分块渲染 + 思考过程展开 | Agent 推理过程可视化 |
| **Perplexity** | 引用来源卡片 + 搜索结果实时展示 | RAG 检索结果内联展示 |
| **Vercel AI SDK** | `useChat` hook + 流式组件 + 工具调用 UI | ai-elements-vue 组件库 |
| **LangGraph Studio** | 节点执行可视化 + 状态机调试 | DAG 执行面板 |
| **Dify** | 可视化工作流 + 节点配置面板 | 已有 X6 画布 |

### 4.2 推荐引入的 AI UI 组件

| 组件 | 来源 | 用途 |
|------|------|------|
| `ChatMessage` | ai-elements-vue | 统一的消息气泡组件（支持 Markdown/代码/工具调用） |
| `ToolCallDisplay` | ai-elements-vue | 工具调用的展开/折叠展示 |
| `SourceCard` | 自建 | RAG 检索来源卡片（文档名/分数/片段预览） |
| `MemoryBadge` | 自建 | 记忆评分徽章（similarity/decay/importance） |
| `ApprovalDialog` | 自建 | 审批节点弹窗（通过/拒绝/超时） |

### 4.3 流式渲染优化

**当前**：使用 `@microsoft/fetch-event-source` + SSE

**优化**：
1. 统一为 `useSSE` composable（消除重复代码）
2. 支持取消（AbortController）
3. 支持重连（指数退避）
4. Markdown 渲染优化：流式块级渲染（不等全部到齐再渲染）

## 五、性能优化

### 5.1 虚拟滚动

**场景**：长对话列表（100+ 消息）、大工具列表、大知识库列表

**方案**：引入 `vue-virtual-scroller`（10.7k⭐）

```vue
<RecycleScroller :items="messages" :item-size="80">
  <template #default="{ item }">
    <ChatMessage :message="item" />
  </template>
</RecycleScroller>
```

### 5.2 代码分割

**当前**：Vite 手动 chunk splitting

**优化**：
- 路由懒加载（已有）
- 大型可视化库（Monaco/G6/X6）按需加载
- Element Plus tree-shaking（已有 unplugin-vue-components）

### 5.3 图片优化

- 大图懒加载（`loading="lazy"`）
- SVG 图标按需加载（已有 vite-plugin-svg-icons）

## 六、设计规范（基于 frontend-design skill）

### 6.1 视觉方向

qKnow 是**企业级 AI 平台**，视觉方向应为：
- **Industrial + Editorial**：专业、可信、信息密度高
- 避免"AI 毛玻璃渐变"等潮流设计
- 强调数据可视化的清晰度
