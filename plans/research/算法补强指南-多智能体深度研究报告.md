## 二、SOTA 补强集成建议书

### 2.1 RAG 引擎补强

#### 2.1.1 Contextual Retrieval（Anthropic, 2024）— P0

**来源**: Anthropic 官方博客 + Claude Cookbook（9 个代码库实测）

**核心思想**: 在 embedding 前，用 LLM 为每个 chunk 生成上下文描述并 prepend。

**关键数据**:

| 方法 | Pass@5 | Pass@10 | Pass@20 |
|------|--------|---------|---------|
| 基线 RAG | 80.92% | 87.15% | 90.06% |
| + Contextual Embeddings | 88.12% | 92.34% | 94.29% |
| + Reranking | 92.15% | 95.26% | **97.45%** |

**成本**: 利用 prompt caching，$1.02 per million document tokens（一次性）

**qKnow 集成方案**:
```java
// 在 KmcSyncServiceImpl 文档同步时注入上下文
@Component
public class ContextualEnrichmentService {
    private final IChatClientService chatClientService;
    
    public String generateChunkContext(String fullDocument, String chunkContent) {
        String prompt = """
            <document>%s</document>
            Here is the chunk we want to situate within the whole document:
            <chunk>%s</chunk>
            Please give a short succinct context to situate this chunk within the overall document.
            Return ONLY the context, nothing else.
            """.formatted(fullDocument, chunkContent);
        return chatClientService.getChatClient().prompt(prompt).call().content();
    }
}
```

**集成位置**: `RecursiveSplitter` 切分后、`EmbeddingServiceImpl` 向量化前

#### 2.1.3 Adaptive-RAG 三级路由（NAACL 2024）— P1

**来源**: [arxiv.org/abs/2403.14403](https://arxiv.org/abs/2403.14403)

**核心思想**: 查询复杂度驱动的三级路由

| 复杂度 | 策略 | 处理方式 |
|--------|------|---------|
| 简单 | No Retrieval | LLM 直接回答 |
| 中等 | Single-step RAG | 标准检索→生成 |
| 复杂 | Multi-step RAG | 迭代检索+推理 |

**关键数据**: +2.8% EM, +3.4% F1, 推理延迟降低 28%

**qKnow 集成方案**:
```java
@Component
public class AdaptiveRagRouter {
    private final QueryComplexityClassifier classifier; // DistilBERT 66M, ~10ms
    
    public RetrievalStrategy route(String query) {
        ComplexityLevel level = classifier.classify(query);
        return switch (level) {
            case SIMPLE -> RetrievalStrategy.NO_RETRIEVAL;
            case MEDIUM -> RetrievalStrategy.SINGLE_STEP;
            case COMPLEX -> RetrievalStrategy.MULTI_STEP;
        };
    }
}
```

**集成位置**: `RagRetrievalService.retrieve()` 入口处

#### 2.1.4 LLMLingua-2 上下文压缩（ACL 2024）— P1

**来源**: [arxiv.org/abs/2403.12968](https://arxiv.org/abs/2403.12968)

**核心突破**: BERT-level token classification，比 LLMLingua 快 3-6×

**关键数据**:
- 压缩率: 15×（保留 20-50% token）
- 精度保持: 优于 LLMLingua
- 速度: 3-6× 更快

**qKnow 集成方案**:
```java
@Component
public class ContextCompressor implements DocumentPostProcessor {
    private final WebClient linguaClient; // LLMLingua-2 HTTP service
    
    @Override
    public List<Document> process(Query query, List<Document> documents) {
        return documents.stream()
            .map(doc -> {
                String compressed = linguaClient.post()
                    .bodyValue(new CompressRequest(doc.getText(), 0.3))
                    .retrieve().bodyToMono(String.class).block();
                return new Document(compressed, doc.getMetadata());
            }).toList();
    }
}
```

**集成位置**: `RagRerankService.rerank()` 后、送入 LLM 前

#### 2.2.2 三级 Agent Router — P0

**来源**: MasRouter (ACL 2025) | NVIDIA LLM Router | QueryComplexityRouter

**核心思想**: 根据查询复杂度自动路由到不同处理路径

| 路由级别 | 处理方式 | 延迟 | 成本 |
|---------|---------|------|------|
| Simple | LLM 直接回答（无检索） | ~100ms | 最低 |
| Direct | 单步 RAG + ReAct | ~500ms | 中等 |
| Complex | Plan-and-Solve + 多步推理 | ~2s | 最高 |

**qKnow 集成方案**:
```java
@Component
public class AgentRouter {
    private final QueryComplexityClassifier classifier; // DistilBERT 66M
    
    public AgentRoute route(String query) {
        ComplexityLevel level = classifier.classify(query);
        return switch (level) {
            case SIMPLE -> AgentRoute.DIRECT_LLM;
            case MEDIUM -> AgentRoute.REACT_AGENT;
            case COMPLEX -> AgentRoute.PLAN_AND_SOLVE;
        };
    }
}
```

**集成位置**: `AgentOrchestrator.chat()` 入口处

#### 2.2.4 fail-plausible 检测 — P2

**来源**: [arxiv 2606.14589](https://arxiv.deeppaper.ai/papers/2606.14589v1)（2026-06）

**核心发现**: LLM Agent 最危险的静默失败模式——LLM 将错误转化为流畅合理叙述欺骗观察者。~70% 静默失败靠人工发现。

**qKnow 应对**: 在 `HermesKernel.reflect()` 中增加 self-consistency 检测：
```java
// 多次采样对比，检测流畅但不一致的回答
List<String> samples = IntStream.range(0, 3)
    .mapToObj(i -> chatModel.call(prompt).getResult().getOutput().getText())
    .toList();
double consistency = calculateSelfConsistency(samples);
if (consistency < 0.7) {
    // 触发人工审核或降级处理
}
```

### 2.3 记忆系统补强

#### 2.3.1 CrewAI 复合评分记忆 — P1

**来源**: CrewAI 官方文档（交叉验证 3-0）

**核心公式**:
```
composite = semantic_weight × similarity + recency_weight × decay + importance_weight × importance
decay = 0.5^(age_days / half_life_days)
```

**默认权重**: semantic=0.5, recency=0.3, importance=0.2, half_life=30 天

**qKnow 集成方案**:
```java
@Component
public class EnhancedMemoryRetriever {
    public List<MemoryEntry> retrieve(String query, String sessionId, int topK) {
        List<MemoryEntry> candidates = longTermMemory.getAll(sessionId);
        return candidates.stream()
            .map(entry -> {
                double semantic = embeddingSimilarity(query, entry.getContent());
                double recency = Math.pow(0.5, entry.getAgeDays() / 30.0);
                double importance = entry.getImportanceScore();
                double composite = 0.5 * semantic + 0.3 * recency + 0.2 * importance;
                return new ScoredMemory(entry, composite);
            })
            .sorted(Comparator.reverseOrder())
            .limit(topK)
            .map(ScoredMemory::entry)
            .toList();
    }
}
```
