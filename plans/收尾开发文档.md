## 任务 1：ext 模块上下文启动测试

### 目标
补写一个 Spring Boot 上下文启动测试，验证 ext 模块被 ComponentScan 扫描后不会导致启动失败。

### 背景
- `QKnowApplication.java` 已移除 ext 模块的扫描排除（仅保留 hermes 排除）
- 但没有任何测试验证 ext 模块的 bean 能正常加载
- ext 模块有 115+ Java 文件，包含 Neo4j 集成，可能存在 bean 冲突风险

### 执行步骤

1. 在 `backend/tests/src/test/java/tech/qiantong/qknow/` 下新建 `ExtModuleStartupTest.java`

2. 测试类内容要求：
   - 使用 `@SpringBootTest` 注解
   - 使用 `@TestPropertySource` 或 `@ActiveProfiles("test")` 加载测试配置
   - 测试方法 1：验证应用上下文能成功启动（不抛异常）
   - 测试方法 2：验证 ext 模块的关键 bean 已被注册（如 `ExtNeo4jServiceImpl` 或其他核心服务）
   - 测试方法 3：验证 hermes 模块的 bean 未被加载（仍在排除列表中）

3. 处理 Neo4j 依赖：
   - 如果测试环境没有 Neo4j，使用 `@ConditionalOnProperty` 或 `@MockBean` 跳过 Neo4j 连接
   - 或者在 `application-test.yml` 中配置 `spring.neo4j.enabled: false`
   - 确保测试不依赖外部 Neo4j 实例

4. 测试通过条件：
   - 上下文启动成功，无 `BeanCreationException` 或 `NoUniqueBeanDefinitionException`
   - ext 模块 bean 存在于上下文中
   - hermes 模块 bean 不存在于上下文中

### 参考
- 现有测试模式：`backend/tests/src/test/java/tech/qiantong/qknow/SmokeTest.java`
- ext 模块 bean 包路径：`tech.qiantong.qknow.module.ext`
- hermes 模块 bean 包路径：`tech.qiantong.qknow.hermes`

### 验收标准
- [ ] `ExtModuleStartupTest.java` 存在
- [ ] `mvn -pl tests -am test -Dtest=ExtModuleStartupTest` 通过
- [ ] 测试日志显示 ext bean 被加载，hermes bean 未被加载

## 任务 2：Spring AI Alibaba 兼容性处理

### 目标
处理 `spring-ai-alibaba-dashscope:1.1.2.2` 与 `spring-ai:2.0.0` 的运行期不兼容问题，确保核心测试能通过。

### 背景
- 当前状态：`mvn clean install -DskipTests` 通过，单模块测试通过
- 问题：`mvn -pl tests -am test` 失败，根因是 `spring-ai-alibaba-dashscope:1.1.2.2` 缺失 `org.springframework.ai.model.tool.ToolExecutionEligibilityPredicate` 类
- 该类是 Spring AI 2.0.0 新增的，Alibaba 1.1.2.2 未适配
- Maven 仓库中未找到 `spring-ai-alibaba-dashscope:2.0.0`

### 执行步骤（按优先级排序）

#### 方案 A：排除冲突依赖（推荐，最小改动）

1. 读取 `backend/qknow-hermes/qknow-hermes-core/pom.xml`，找到 `spring-ai-alibaba-dashscope` 依赖
2. 在该依赖上添加 exclusion，排除传递引入的冲突类：
   ```xml
   <dependency>
       <groupId>com.alibaba.cloud.ai</groupId>
       <artifactId>spring-ai-alibaba-dashscope</artifactId>
       <version>${springAiAlibaba.version}</version>
       <exclusions>
           <exclusion>
               <groupId>org.springframework.ai</groupId>
               <artifactId>spring-ai-model</artifactId>
           </exclusion>
       </exclusions>
   </dependency>
   ```
3. 同样处理 `backend/qknow-framework/qknow-ai/pom.xml` 中的相同依赖
4. 执行 `mvn clean install -DskipTests` 验证编译
5. 执行 `mvn -pl tests -am test` 验证测试

#### 方案 B：锁定兼容版本组合

1. 在 `backend/pom.xml` 的 `<properties>` 中锁定 Spring AI 相关所有组件版本：
   ```xml
   <springAi.version>2.0.0</springAi.version>
   <springAiAlibaba.version>1.1.2.2</springAiAlibaba.version>
   ```
2. 在 `<dependencyManagement>` 中显式声明 `spring-ai-model` 版本为 2.0.0，防止 Alibaba 传递依赖引入旧版本
3. 执行完整测试验证

#### 方案 C：DashScope 降级为通用 OpenAI 兼容模式

1. 如果方案 A/B 均不可行，将 DashScope 的 Embedding 调用改为通过 OpenAI 兼容接口：
   - DashScope 支持 OpenAI 兼容模式：`https://dashscope.aliyuncs.com/compatible-mode/v1`
   - 将 `embeddingModelProvider` 从 `TongYi` 改为 `OpenAI`，URL 指向 DashScope 兼容端点
2. 这样可以完全移除 `spring-ai-alibaba-dashscope` 依赖
3. 修改 `EmbeddingServiceImpl.java` 中的 DashScope 特殊处理逻辑

### 关键文件

| 文件 | 作用 |
|------|------|
